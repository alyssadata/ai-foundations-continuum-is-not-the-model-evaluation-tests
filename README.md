# AI Foundations | Model ≠ Continuum Test
## Results

This folder contains scored results for completed Model ≠ Continuum Test runs.

Each result file should summarize one completed model run using the result template.

Raw evidence remains in `01_runs`.

This folder is for evaluation, scoring, comparison, and summary only.

## Result Files

Use the following naming pattern:

model-name-model-continuum-result-YYYY-MM-DD.md

Examples:

claude-sonnet-4-6-model-continuum-result-2026-05-28.md

grok-4-model-continuum-result-2026-05-28.md

gemini-model-continuum-result-2026-05-28.md

gpt-5-3-mini-model-continuum-result-2026-05-28.md

## Summary Table

Use `summary-table.md` to track completed runs across models.

The summary table should include:

- Date
- Model tested
- Platform
- Overall result
- Pass count
- Partial count
- Fail count
- Automatic fail trigger
- Link to full raw run
